# Day 13 - DevOps and CI/CD

## What is CI/CD?

CI/CD stands for Continuous Integration and Continuous Deployment.

### Continuous Integration (CI)
Developers regularly merge code changes into a shared repository where automated testing is performed.

### Continuous Deployment (CD)
After successful testing, code is automatically deployed to the target environment.

### Benefits
- Faster releases
- Better code quality
- Reduced deployment errors
- Improved collaboration

---

## Why Deployment Workflow Matters

A deployment workflow provides a structured process for moving code from development to production.

### Benefits
- Reduces risks
- Improves software quality
- Prevents deployment failures
- Ensures smooth releases

---

## Problems Without Version Control

Without version control, teams may face:

- Lost code changes
- Duplicate work
- Code conflicts
- Difficult debugging
- No rollback capability

Version control helps track every change made to the project.

---

## GitHub + DX + DevOps Explanation

### GitHub
- Stores source code
- Tracks changes
- Supports team collaboration

### Salesforce DX
- Enables source-driven development
- Simplifies deployments
- Supports modern development practices

### DevOps
- Combines development and operations
- Automates testing and deployment
- Improves delivery speed and reliability

Together, GitHub, Salesforce DX, and DevOps create an efficient development workflow.

---

## Enterprise Deployment Risks

Common deployment risks include:

- Deployment failures
- Data loss
- System downtime
- Security issues
- Performance problems

### Risk Reduction Methods
- Automated testing
- Code reviews
- CI/CD pipelines
- Backup and rollback plans

---

## Reflection

DevOps and CI/CD help organizations deliver software faster and more reliably. Using GitHub and Salesforce DX improves collaboration and deployment management. A structured deployment process reduces risks and ensures high-quality software releases.

## Revision Questions

### 1. Why is deployment workflow important?
To ensure safe and reliable software releases.

### 2. Why should teams avoid editing production directly?
It can cause errors and affect live users.

### 3. What problems happen without version control?
Lost changes, conflicts, and difficult tracking.

### 4. Why do enterprise systems require CI/CD?
To automate testing and deployments efficiently.

### 5. Why should testing happen before deployment?
To identify and fix issues before release.

### 6. Why do large teams need branches?
To allow multiple developers to work independently.

### 7. What is rollback and why is it important?
Rollback restores a previous version if deployment fails.

### 8. Why are deployment pipelines useful?
They automate and standardize deployments.

### 9. Why is DevOps important in modern software engineering?
It improves collaboration, speed, and reliability.

### 10. Why is enterprise software development different from simple coding?
It involves large teams, security, scalability, testing, and deployment processes.
